# 💅 NailVirtuoso

**NailVirtuoso** is a full-stack AI-powered nail art marketplace and design generator built with Next.js. Users can browse curated nail art templates, upload their own designs for review, and generate personalized nail art from a photo of their hand using AI. The platform includes a comprehensive **Supreme Admin Console** (`/admin`) that gives administrators full control over content, users, tags, and system health.

---

## ✨ Features

### For Users
- **Marketplace** – Browse all approved nail art templates with tag-based filtering.
- **AI Generation** – Upload a photo of your hand and apply any template to see a realistic preview generated by AI (Hugging Face FLUX.1 + Google Gemini).
- **Save Favourites** – Bookmark templates to your personal saved collection.
- **Authentication** – Sign in with Google OAuth or email/password via Firebase Auth.

### For Editors
- **Creator Studio** (`/editor`) – Submit new nail art templates (marketplace image + reference image + tags) for admin review.

### For Admins
- **Supreme Admin Console** (`/admin`) – Full platform control (see section below).

## 🛡️ Admin Panel — Full Control

The admin panel is accessible only to users with `role: "admin"` stored in Firestore. Any other user trying to visit `/admin` is immediately redirected to the homepage.

### Access Guard
```typescript
// app/admin/page.tsx
if (!loading && (!appUser || appUser.role !== 'admin')) {
    router.push("/");
}
```

### Navigation Tabs

The console is split into **five tabs**, each giving the admin a different dimension of control:

---

#### 📊 Status Tab
Real-time system health dashboard.

| Widget | Description |
|--------|-------------|
| **Gemini Quota Meter** | Tracks daily AI generations against the 1,500/day free-tier limit. Progress bar turns red above 85%. |
| **API Health Card** | Calls `/api/admin/health-check` to verify Gemini and Hugging Face (FLUX.1) are reachable. Shows `active` (black), `zeroed` (orange), or `error` (red). Includes a **Re-Diagnose** button. |
| **Inventory Counter** | Total number of approved templates in the marketplace. |
| **Pending Counter** | Templates waiting for admin review. |
| **Mortals Counter** | Total registered users. |
| **Trends Counter** | Tags discovered in user submissions that are not yet official main spheres. |

---

#### ⚡ Queue Tab
Template moderation queue — review every submission before it goes live.

- Lists all templates with `status: "pending"`.
- Displays the marketplace image, title, and submitting editor's name.
- **Approve** button → sets `status: "approved"`, template appears in the marketplace immediately.
- **Reject** button → sets `status: "rejected"`, template is hidden from users.
- When the queue is empty, displays *"The Queue is at peace."*

---

#### 🗂️ Inventory Tab
Full vault of every approved template.

- Grid view (5 columns) of all `status: "approved"` templates.
- Hover over any card to reveal a red **Delete** (🗑️) button.
- Deletion is permanent and requires a browser confirmation dialog.
- Empty state: *"The Vault is empty."*

---

#### 🌐 Spheres Tab
Manage the official tag categories ("Spheres") used to filter templates.

| Control | Description |
|---------|-------------|
| **Create Sphere form** | Enter a sphere name and a background image URL, then click **Manifest** to add it to Firestore (`tags` collection, `isMain: true`). |
| **Sphere grid** | Circular thumbnail cards for every main tag. |
| **Delete Sphere** | Hover over a sphere card — it turns red with a trash icon. Click to permanently delete the tag from Firestore. |
| **Trends discovery** | Tags that users have already applied to templates but that are not yet official spheres appear in the **Trends** counter on the Status tab, helping admins decide which new spheres to create. |

---

#### 👥 Users Tab
Full user roster with role management.

- Table listing every registered user with their **username**, **email**, and current **role badge**.
- Role badges: `admin` (red) vs. any other role (grey).
- **Promote** button → sets `role: "editor"`, granting access to the Creator Studio.
- **Revoke** button → sets `role: "user"`, removing editor access.
- Admins cannot be demoted through this interface (only via direct Firestore edits).

---

### Admin API Routes

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/admin/health-check` | `GET` | Pings Gemini Flash and Hugging Face FLUX.1; returns `{ status, message }` for each. Returns `429` if quota is exhausted. |
| `/api/admin/generate-meta-prompt` | `POST` | Accepts a template image URL, uses Gemini to analyse it, and returns an optimised AI meta-prompt for the generation pipeline. |

---

### Template Lifecycle

```
Editor submits  →  [pending]  →  Admin approves  →  [approved]  →  Visible in marketplace
                              ↘  Admin rejects   →  [rejected]  →  Hidden from users
                              ↘  Admin deletes   →  Permanently removed
```

## 👤 Role System

| Role | Capabilities |
|------|-------------|
| `admin` | Full access to Supreme Admin Console + Creator Studio |
| `editor` | Access to Creator Studio (`/editor`) to submit templates |
| `user` | Browse marketplace, save templates, run AI generation |

---

## 🗄️ Data Models (Firestore)

| Collection | Key Fields |
|-----------|-----------|
| `users` | `uid`, `email`, `username`, `photoURL`, `role`, `createdAt` |
| `templates` | `id`, `title`, `marketplaceImage`, `referenceImage`, `tags[]`, `status`, `editorName`, `createdBy`, `adminComment`, `createdAt` |
| `tags` | `id`, `name`, `bgImage`, `isMain` (boolean), `createdAt` |
| `generations` | `createdAt` (timestamp) — one document per generation, used for quota tracking |

NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=

GEMINI_API_KEY=
HF_TOKEN=

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- npm, yarn, pnpm, or bun

### Install & Run

```bash
# Install dependencies
npm install

# Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Other Commands

```bash
npm run build   # Production build
npm start       # Run the production server
npm run lint    # Run ESLint
```

---

## 🌐 Deployment

Deploy to [Vercel](https://vercel.com) with zero configuration — just connect the repository and add your environment variables in the Vercel dashboard.

```bash
npx vercel --prod
```

For other platforms, run `npm run build` and serve the `.next` directory.
